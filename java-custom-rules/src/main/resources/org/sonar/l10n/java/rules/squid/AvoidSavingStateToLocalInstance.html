Avoid saving session or other state information to a local instance. Application nodes Running in a containerized environment are ephemeral and can go away at any time without warning. Any state information should be written to an external store, such as a database or an in memory cache platform that allows for persisting state (e.g. Redis).

Noncompliant Code Example

public class SavingStateExample {

    public SavingStateExample() { 
      FileOutputStream example1 = new FileOutputStream("/home/shoppingcart.txt"); // Noncompliant
	  FileWriter example2 = new FileWriter("/home/shoppingcart.txt"); // Noncompliant  
      BufferedWriter example3 = new BufferedWriter("/home/shoppingcart.txt"); // Noncompliant
    }
    
}
